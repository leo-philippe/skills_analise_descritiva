Fundamentos

3 metodologias acadêmicas integradas

O pipeline integra três metodologias consolidadas, referenciadas em literatura científica, aplicadas em sequência lógica.
1
Data Profiling — "Antes de analisar, apresente-se aos seus dados"
Rahm & Do (2000) — IEEE Data Engineering Bulletin
Imagine que você chegou em um trabalho novo e precisa liderar uma equipe que nunca viu antes. Você não vai distribuir tarefas antes de saber quem são essas pessoas, o que cada uma faz e se alguma está faltando — certo? Com os seus arquivos de dados é exatamente a mesma coisa.

Abra os arquivos e responda estas perguntas — uma coluna de cada vez:

  • Que tipo de informação é essa? — número, texto, data, código? Saber o tipo evita análises erradas desde o início.
  • Tem campos em branco? — se sim, quantos e onde? Um campo 80% vazio pode ser inútil para a análise.
  • Os valores fazem sentido? — uma idade de 300 anos, um CPF com letras ou uma venda negativa são alertas vermelhos.
  • Os dados têm um período claro? — de quando a quando vai o histórico? Isso define o escopo de tudo que vem depois.
Rahm, E. & Do, H.H. (2000). Data Cleaning: Problems and Current Approaches. IEEE Data Engineering Bulletin, 23(4), 3–13.
2
EDA — "Converse com os dados. Deixe eles te surpreenderem."
Tukey (1977) · De Mast & Trip (2007) — Journal of Quality Technology
Pense em uma conversa de verdade: você faz uma pergunta, ouve a resposta, e a resposta te faz pensar em uma nova pergunta. Ninguém entra em uma conversa com o roteiro completo escrito — você vai descobrindo no caminho. A EDA funciona exatamente assim.

O ciclo criado por Tukey se repete quantas vezes forem necessárias:

Olhe os dados
→
Encontre algo interessante
→
Interprete o que viu
→
Nova pergunta
↩ repete

Use os gráficos e métricas do output como ferramentas dentro desse ciclo: cada visualização é uma forma diferente de olhar para os dados e provocar uma nova pergunta de negócio.

Tukey, J.W. (1977). Exploratory Data Analysis. Addison-Wesley.
3
Framework U-B-M — "Do simples ao complexo. Nunca pule etapas."
Peat & Barton (2005) — Medical Statistics; PMC/NCBI
Aprender a cozinhar tem uma ordem: primeiro você entende cada ingrediente sozinho. Depois aprende como eles se combinam. E só então monta um prato completo. Pular para o prato sem entender o ingrediente é a receita do desastre.
Nível 1
Univariada
"O que é essa variável?"
Uma coluna de cada vez — média, mediana, distribuição, nulos.
Nível 2
Bivariada
"Essas duas têm relação?"
Dois campos juntos — scatter, correlação, cross-tab.
Nível 3
Multivariada
"O que tudo mostra junto?"
Várias colunas — heatmap, bubble chart, visão completa.

A regra de ouro: nunca vá para o nível 2 sem ter terminado o nível 1. Relações entre variáveis que você não entende individualmente levam a conclusões erradas.

Peat, J. & Barton, B. (2005). Medical Statistics: A Guide to Data Analysis and Critical Appraisal. Blackwell.
